# Ordino 🍽️

**Ordino** (от ит. *ordinare* — „поръчвам" / „подреждам") е уеб базирана система за управление на ресторант, която обединява в едно всички аспекти от ресторантьорския бизнес — рецепти, менюта, склад и потребители.

---

## Описание

Повечето ресторанти разчитат на комбинация от няколко различни софтуера и физически носители на информация, за да проследяват работните си процеси. Ordino решава този проблем, като предоставя единна платформа, която обхваща целия жизнен цикъл на рецептите и менютата, управлението на склада и потребителите — с наблег върху удобство, прозрачност и лесно търсене на отговорност.

## Архитектура

Системата е изградена като **монолит с ясни модулни разделения**, контейнеризиран с Docker (приложението и базата данни са в отделни контейнери). Проектирана е така, че лесно да се мигрира към облачен хостинг (Google Cloud, AWS) с поддръжка на автоматично архивиране и point-in-time recovery.

```
┌─────────────┐       REST API      ┌─────────────────┐       ORM (Hibernate)      ┌───────────┐
│  Vue.js SPA │ ──────────────────▶ │  Spring Boot    │ ─────────────────────────▶ │   MySQL   │
│ (TypeScript)│                     │  (монолит)      │                            │           │
└─────────────┘                     └─────────────────┘                            └───────────┘
```

**Модули на бекенда:**
- Управление на потребители
- Рецепти
- Менюта
- Склад

## Роли и права

### Администратор
- Добавя, редактира и изтрива потребители
- Задава роли и права
- Нулира пароли

### Готвач
- Създава и редактира рецепти (чернови)
- Предава рецепти за одобрение
- Преглежда одобрени и архивирани рецепти и историята им
- Експортира рецепти
- Преглежда менюта

### Главен готвач *(разширени права)*
- Одобрява или връща рецепти за преработка
- Редактира всички одобрени рецепти
- Архивира рецепти и ги връща от архива
- Управлява категории на рецепти, мерни единици и категории продукти
- Създава и редактира менюта
- Добавя снимки към рецепти

### Кухненски персонал
- Преглежда одобрени рецепти и менюта

### Складов отговорник
- Управлява доставки, доставчици и инвентар
- Управлява продукти и категории продукти за склада
- Получава известия (изтичащи продукти, ниска наличност, нови продукти в одобрени рецепти)
- Преглежда справки за склада

### Управител
- Преглежда всички модули в read-only режим

## Основни функционалности

### Рецепти
- Пълен жизнен цикъл: чернова → одобрение → архив
- История на редакциите с diff view и снапшоти
- WYSIWYG редактор за инструкции за приготвяне
- Калкулатор на цената на рецептата
- Снимки към рецепти
- Търсене по име, продукти и бележки; филтриране по категория

### Менюта
- Менюта с секции и наредени рецепти

### Склад
- Управление на доставки и доставчици
- Инвентаризация по партиди с дата на изтичане
- Отписване на продукти с причина за загуба
- Справки: разходи по време, причини за загуба, разлики очаквано/получено, най-поръчвани продукти
- Dashboard: продукти под минимална наличност, изтичащи срокове, очаквани доставки

### Потребители
- Автентикация с потребителско име и парола
- Задължителна смяна на парола при първо влизане
- Soft delete на потребители
